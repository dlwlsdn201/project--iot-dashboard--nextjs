# [PRD] SmartEnergy BEMS Dashboard

## 1. Product Overview
- **Repository**: `project--iot-dashboard--nextjs`
- **System Context**: Embedded IoT Sensor -> Server -> Frontend Dashboard.
- **Objective**: 구축용 BEMS/FEMS(건물/공장 에너지 관리 시스템) 환경에 적합한 '고밀도 데이터 관제 대시보드' 프론트엔드 구현. 
- **Core Values**: 
  1. 단순한 SaaS 스타일을 배제한 전문적인(Trustworthy) 산업용 관제 UI.
  2. 한 화면에 방대한 데이터를 압축적으로 보여주는 고밀도 그리드(Bento-box) 레이아웃.
  3. 실시간 데이터 스트리밍 환경에서의 렌더링 부하 최적화.

## 2. Target Audience & Scenarios
- **Primary User**: 건물/공장 시설 관리자 및 에너지 분석가.
- **Pain Point**: 데이터가 여러 화면에 흩어져 있어 이상 징후 파악이 늦음.
- **Solution**: 7개의 핵심 위젯을 한 화면(12-column grid)에 조립하여, 스크롤 이동을 최소화하고 직관적인 모니터링 환경 제공.

## 3. Core Features (The 7 Dashboard Widgets)
아래 7개의 위젯(W1~W7)을 12-column 그리드 상에 배치하는 것을 핵심으로 한다.
- **[W1] 실시간 에너지 스트리밍 차트 (Live Energy)**: 초 단위 전력 사용량 라인 차트.
- **[W2] 최근 7일 에너지원별 추세 (7-Day Trend)**: 전기, 가스, 열 등 다중 에너지원 누적 막대 차트.
- **[W3] 단위별 탄소 배출량 (Carbon Emission)**: 주/월/년 필터 탭이 포함된 막대 차트.
- **[W4] 실시간 및 피크 부하 (Load Gauge)**: 실시간 부하 수치와 일일 최대 부하를 직관적으로 보여주는 반원형 게이지(Gauge) UI.
- **[W5] 실시간 이벤트 로그 (Event Alerts)**: 위험(Danger), 경고(Warning) 등 심각도별 색상이 적용된 스크롤 가능한 최신 이벤트 리스트.
- **[W6] 전년 동월 대비 사용량 (YoY Comparison)**: 작년(Line)과 올해(Bar)를 겹쳐서 비교하는 콤보 차트.
- **[W7] 5년 주기 전력 사용 예측 (5-Year Prediction)**: 과거 실선 데이터와 미래 점선(Dotted) 예측 구간이 포함된 라인 차트.

### 3.1 5년 주기 전력 사용 예측 (5-Year Prediction)** 위젯 목적 및 상세 의도

1. 목적 및 데이터 관점 (Why & What)
사용자(시설 관리자, 경영진)가 이 데이터를 보는 이유:

전력 계약 및 예산 수립 (CAPEX/OPEX): 공장이나 대형 건물은 한국전력 등과 '최대 수요 전력(Peak)'을 기준으로 연간 계약을 맺습니다. 향후 1~5년간의 전력 사용량 증가 추이를 예측해야 내년도 에너지 예산을 미리 편성하고, 변압기 증설이나 태양광 패널 설치 같은 대규모 투자를 결정할 수 있습니다.

노후화 설비 탐지: 생산량이나 입주 인원이 비슷한데 5년 추세선이 꾸준히 우상향한다면, HVAC(공조기)나 모터 설비가 노후화되어 에너지 효율이 떨어지고 있다는 강력한 증거가 됩니다.

ESG 및 탄소 중립(Net-Zero) 목표 트래킹: 기업의 장기적인 탄소 감축 목표치(Target Line)와 실제 예측치(Predicted Line)가 일치하게 가고 있는지 추적합니다.

데이터 구조 (Mocking Data 설계):

X축 (시간): 과거 4년 + 현재 연도 + 미래 1년 (총 5~6년치의 월별/연도별 데이터).

Y축 (사용량): 전력 사용량 (MWh 또는 GWh).

데이터 속성: 객체 내에 actual (실제 사용량)과 predicted (예측 사용량) 값이 분리되어 있거나, isForecast: boolean 플래그로 과거와 미래 데이터를 구분해야 합니다.

2. UI/UX 디자인 관점 (How it looks)
이 차트는 과거의 '확정된 팩트'와 미래의 '불확실한 예측'을 시각적으로 완벽하게 분리하는 것이 핵심입니다.

실선과 점선의 결합 (Solid & Dashed Lines):

과거~현재 구간: Cyan-400이나 Blue-500 색상의 **굵은 실선(Solid Line)**으로 렌더링하여 신뢰성 있고 확정된 데이터를 표현합니다.

미래 예측 구간: 동일한 선형을 이어가되, 색상의 채도를 살짝 낮추거나(Slate-400 또는 Amber-500) **점선(Dotted/Dashed Line, CSS의 stroke-dasharray 활용)**으로 처리하여 "아직 발생하지 않은 예측값"임을 사용자가 직관적으로 인지하게 합니다.

'Today' 구분선 (Vertical Reference Line):

현재 시점(오늘/이번 달)의 X축 위치에 세로점선(Reference Line)을 긋고 "Current" 또는 "Today"라는 라벨을 붙여 과거와 미래의 경계를 명확히 가릅니다.

신뢰 구간 (Confidence Interval / Shaded Area): (고급 UI)

미래 예측 선 위아래로 반투명한 면적(Area, fill-slate-700/30)을 그려 넣습니다. 예측에는 오차가 존재하므로 "최소 이만큼에서 최대 이만큼 쓸 것이다(예: 95% 신뢰구간)"라는 밴드를 보여주면 산업용 대시보드로서의 전문성이 극대화됩니다.

3. 프론트엔드 포트폴리오(면접) 어필 포인트 💡
나중에 이 위젯을 다 만들고 나서 포트폴리오 README나 면접에서 이렇게 설명하시면 매우 좋은 평가를 받을 수 있습니다.

"5년 전력 예측 차트(W7)를 구현할 때, 과거 데이터와 예측 데이터를 하나의 연속된 라인으로 그리면서도 시각적 스타일(실선 -> 점선)을 전환하는 데 기술적 고민을 했습니다.

Recharts(또는 EvilCharts)에서 하나의 <Line /> 컴포넌트는 단일 스타일만 적용되기 때문에, 데이터 배열을 가공하여 과거 데이터용 선과 예측 데이터용 선 두 개를 겹쳐 그리고, 두 선이 끊어지지 않고 부드럽게 이어지도록 데이터 포인트를 교차 연결(Intersection)하는 데이터 파싱 로직을 구현했습니다. 이를 통해 사용자가 과거의 팩트와 미래의 예측을 인지적 부하 없이 한눈에 구분할 수 있도록 UX를 개선했습니다."

Claude 에이전트에게 이 위젯을 만들라고 지시하실 때, **"W7 차트는 반드시 현재 시점을 기준으로 과거는 실선, 미래는 점선(stroke-dasharray)으로 렌더링하고, 현재 시점에 세로 구분선(ReferenceLine)을 넣어줘"**라고 덧붙이시면 훨씬 더 퀄리티 높은 코드를 뽑아낼 수 있을 것입니다!
